The {\ttfamily \mbox{\hyperlink{nostriot__provider_8cpp}{src/nostriot\+\_\+provider.\+cpp}}} module lets you customise your Nostr IoT device to match your own hardware and pricing model. In {\ttfamily \mbox{\hyperlink{nostriot__provider_8cpp}{src/nostriot\+\_\+provider.\+cpp}}} you define what the device can do (capabilities), and how much each action costs (in sats). The example includes an number of example methods with different pricing. This demonstrates how to set up free and variable paid services.

To adapt it, change the hardware configuration for your pins and sensors in the {\ttfamily init()} function, then add or edit entries in capabilities\+\_\+with\+\_\+pricing. Each capability’s behaviour is defined in the {\ttfamily run()} function, where you decide what happens when it’s called.\hypertarget{index_autotoc_md7}{}\doxysubsection{Building and Flashing}\label{index_autotoc_md7}
